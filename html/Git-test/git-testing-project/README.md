# Git Testing Project

This project is designed to test Git's functionality with various file types, including text and HTML files. It includes sample files that can be used to verify basic file handling and version control capabilities.

## Project Structure

```
git-testing-project
├── test-files
│   ├── sample1.txt
│   ├── sample2.txt
│   ├── index.html
│   └── about.html
└── README.md
```

## Purpose

The purpose of this project is to provide a simple and effective way to test Git's ability to track changes in different file formats. The included files serve as examples for testing various Git commands and workflows.

## File Descriptions

- **test-files/sample1.txt**: This file contains sample text for testing purposes. It can be used to verify basic text file handling in Git.

- **test-files/sample2.txt**: This file also contains sample text for testing. It serves a similar purpose as sample1.txt.

- **test-files/index.html**: This file is an HTML document that can be used to test HTML file tracking in Git. It may contain basic HTML structure.

- **test-files/about.html**: This file is another HTML document for testing. It can include additional content or structure to further test Git's handling of HTML files.

## Usage

To use this project for testing Git:

1. Clone the repository to your local machine.
2. Navigate to the project directory.
3. Use Git commands to track changes, commit files, and explore version control features with the provided sample files. 

Feel free to modify the sample files and observe how Git tracks those changes.
